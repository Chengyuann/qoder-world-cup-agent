# 系统架构设计

## 作品定位

本作品是一个世界杯冠军预测 Agent。它不是单纯给出冠军名称，而是把小组赛、淘汰赛、比分预测、冠军概率和推理证据放在同一个可交互页面里，让评审能够看到 Agent 的决策链路。

## 架构分层

1. 数据采集层

   数据层整理球队分组、公开排名、近期状态、阵容深度、淘汰赛经验、攻防能力和北美场地适应度。当前版本使用内置结构化数据表，便于公开部署和复现；后续可接入 FIFA、公开足球数据 API 或自维护 JSON 数据源。

2. 特征工程层

   每支球队被转换为五类核心权重：

   - 综合实力
   - 近期状态
   - 阵容深度
   - 淘汰赛经验
   - 场地适应

   页面提供权重滑杆，评审可以调节参数并观察冠军路径是否稳定。

3. 推理决策层

   Agent 根据权重计算球队综合评分，再叠加攻防错位和杯赛经验修正，逐场生成胜率、比分和解释语句。小组赛采用单循环模拟，淘汰赛采用 32 强到决赛的逐轮推演。

4. 结果输出层

   页面输出冠军候选概率、小组出线表、淘汰赛对阵树、决赛解释和开发工具链证据。所有结论都带有可解释说明。

5. 部署层

   项目是 Vite + React 静态站点，可部署到 GitHub Pages、阿里云 OSS 静态网站、ESA 或函数计算静态托管。

## 当前预测结论

当前模型默认权重下，预测冠军为法国。主要原因是阵容深度、锋线速度、淘汰赛经验和整体攻防质量都处于第一梯队。阿根廷、西班牙、巴西、德国和葡萄牙是主要竞争者。

## 创新点

- 用可调参数把冠军预测从“黑箱答案”变为可验证推理。
- 同时输出赛程树、比分预测和解释语句，而不是只输出概率榜。
- 把“场地适应度”作为北美世界杯特有变量，体现 2026 赛制背景。
- 将提交材料所需的系统架构、Qoder 工具链、公开部署入口集中在页面中，便于评审查看。
