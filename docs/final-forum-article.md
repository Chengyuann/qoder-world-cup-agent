# Qoder码力星期四·世界杯挑战赛 - WorldScope 世界杯冠军预测 Agent

## 作品公开地址

https://chengyuann.github.io/qoder-world-cup-agent/

## 一、作品简介

WorldScope 是一个世界杯冠军预测 Agent。它基于 2026 年 7 月 3 日公开数据快照，先锁定已经发生的小组赛积分和 32 强部分真实赛果，再对尚未结束的淘汰赛路径进行模型推演，输出冠军预测、逐轮比分、胜率和推理依据。

当前默认模型预测冠军为法国。页面不是只展示一个冠军名称，而是把真实数据快照、冠军概率、淘汰赛树、小组积分、决赛解释和权重实验台放在同一个可交互页面中。评审可以调整模型权重或切换情景预设，观察冠军路径如何变化。

## 二、最新数据快照

本次修订重点校正了球队与分组信息，避免出现不在当前正赛分组里的旧占位球队。

- 分组与 48 支球队：按 FIFA / Olympics 公开页面核对。
- 小组积分与晋级状态：参考 NBC Sports 和 WorldCupStats 的 2026 小组最终积分表。
- 32 强赛果：已结束场次锁定真实比分；7 月 3 日尚未结束的澳大利亚 vs 埃及、阿根廷 vs 佛得角、哥伦比亚 vs 加纳继续由模型预测。
- 页面口径：真实赛果与模型预测分开标记，避免把预测结果误写成已发生赛果。

## 三、系统架构设计

Agent 采用五层架构：

1. 数据采集层
   整理官方分组、小组积分、32 强真实赛果、公开排名、阵容状态、球队攻防能力和北美场地适应度，形成结构化数据快照。

2. 特征工程层
   将球队数据转换为综合实力、近期状态、阵容深度、淘汰赛经验、场地适应五类核心特征，并保留攻防能力作为比分预测修正项。

3. 推理决策层
   对真实已完赛场次直接锁定结果；对未完赛比赛使用加权评分、攻防错位修正、Logistic 胜率映射和预期进球模型生成胜负概率与比分。

4. 可解释输出层
   每场关键比赛都展示胜率、比分和解释语句。真实赛果会标记为“真实赛果”，预测场次会标记为“模型预测”。

5. 可视化与部署层
   前端使用 React + TypeScript + Vite 构建，部署到 GitHub Pages，页面可公开访问并交互使用。

## 四、分析决策能力

Agent 默认权重为：

- 综合实力 34
- 近期状态 22
- 阵容深度 18
- 淘汰赛经验 16
- 场地适应 10

评审可以在页面中实时调节权重。每次调整后，系统会重新计算冠军概率、淘汰赛对阵、比分预测和冠军路径。

页面还提供四个情景实验预设：

- 稳态实力模型：重视长期实力和阵容深度。
- 状态爆发模型：提高近期状态权重，观察黑马路径。
- 杯赛经验模型：强调淘汰赛经验和低比分能力。
- 北美适应模型：提高场地适应权重，测试东道主和美洲球队红利。

这些情景可以体现 Agent 不是固定答案，而是可解释、可调参、可复核的预测系统。

## 五、结果输出能力

Agent 输出内容包括：

- 当前预测冠军
- Top 10 冠军候选概率
- 小组赛真实积分与晋级状态
- 32 强真实赛果与未完赛预测
- 16 强、8 强、半决赛和决赛路径
- 每场淘汰赛比分和胜率
- 决赛推理解释
- 冠军路径摘要

默认推演下，页面展示法国夺冠，并给出法国通往决赛的关键路径和决赛比分预测。

## 六、可视化呈现

页面重点优化了可视化和交互体验：

- 首屏冠军卡展示冠军、概率、模型评分和决赛比分。
- SVG 雷达图动态展示冠军球队的五项特征。
- 动态数据流展示 Top 3 概率、决赛比分、真实 32 强赛果数和未完赛预测数。
- 概率排行榜使用进度条展示候选球队梯度。
- 淘汰赛对阵卡明确标记“真实赛果”或“模型预测”。
- 小组积分榜展示真实积分与净胜球，用颜色区分直接晋级和第三名晋级。
- 背景使用轻量 canvas 粒子和连接线，提升动态效果但不影响性能。
- 支持响应式布局和 `prefers-reduced-motion` 动效降级。

## 七、创新与创意

本作品的创新点主要有：

1. 真实赛果和模型预测混合推演
   已发生比赛不再重复预测，模型只负责未完赛路径和后续推演。

2. 可解释权重实验台
   评审可以直接改变模型假设，而不是只看一个固定答案。

3. 情景实验预设
   把“实力优先、状态优先、杯赛经验优先、北美适应优先”做成可点击模型假设。

4. 冠军路径推理
   输出从当前淘汰赛状态到决赛的完整路径，避免单点冠军预测缺少过程。

5. 北美适应度变量
   针对 2026 世界杯举办地特点，把旅行、气候、时差和主场环境纳入模型。
