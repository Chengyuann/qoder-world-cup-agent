## 世界杯冠军预测 Agent — 路演提纲

### 一、赛题理解

- **赛题核心**：构建一个可解释的 Agent，预测 2026 世界杯冠军，需覆盖 48 队、12 小组的扩展赛制。
- **关键挑战**：48 队规模下淘汰赛推演复杂度高；纯规则模型需兼顾准确性与可解释性；前端需实时响应用户参数调整。
- **解题思路**：放弃黑箱 ML，采用"特征加权 + Logistic 胜率 + Softmax 概率分布"的透明管线，所有中间结果可追溯。

### 二、Agent 架构（五层设计）

1. **数据层**：48 队 7 维特征硬编码（strength / form / depth / knockout / travel / attack / defense）。
2. **特征工程层**：加权评分 → Logistic 胜率函数 → 期望进球 → 比分生成，全链路纯函数。
3. **推理层**：小组循环赛模拟 → R32/R16/QF/SF/F 逐级淘汰赛推演，每轮输出中文解释。
4. **概率层**：Softmax（T=4.3）生成 48 队冠军概率分布，含联盟加成。
5. **展示层**：React SPA，单页呈现冠军卡、概率榜、完整赛程树、小组积分榜。

### 三、可解释预测模型

- **胜率公式**：`P = σ(0.16·ΔScore + 0.055·ΔAttack-Defense + 0.035·ΔDefense-Attack)`，系数可审计。
- **用户可调权重**：5 个滑块（0-60），拖动即刻重算全赛程，直观展示"如果加强状态因子，阿根廷如何超越法国"。
- **逐场解释**：每场淘汰赛生成 3 条中文推理依据（综合评分对比、阵容深度、大赛经验），决赛结果直接可溯源。

### 四、可视化页面

- **Hero 区**：冠军预测卡片 + Top-3 概率条 + 元数据条。
- **控制面板**：5 维权重滑块 + 重置按钮，实时联动全局。
- **概率排行榜**：Top-10 候选球队概率条形图。
- **赛程树**：完整 R32→决赛可视化，每场显示比分、胜率、解释。
- **小组积分榜**：12 组 × 4 队，晋级标记高亮。
- **暗色主题**：绿金配色，响应式三档断点，Glassmorphism 卡片风格。

### 五、部署与加分项

- **CI/CD**：GitHub Actions → Vite Build → GitHub Pages 自动部署，零运维。
- **在线地址**：`https://chengyuann.github.io/qoder-world-cup-agent/`
- **技术栈**：React 18 + TypeScript 5.7 + Vite 6 + pnpm，纯静态无后端。
- **加分项**：全链路可解释、用户交互式调参、48 队完整赛制覆盖、中文推理输出、自动化浏览器验证（Playwright）。

### 六、Qoder 工具链使用

- 使用 **Qoder CLI** 完成全部代码生成、架构设计与文档撰写。
- Agent 辅助完成：赛制规则梳理、TypeScript 类型定义、CSS 暗色主题编写、GitHub Actions 配置。
- 利用 Qoder 多轮对话迭代预测模型参数，从初版简单加权到 Logistic + Softmax 方案演进。
- 通过 Qoder 生成架构文档、论坛帖子草稿、证据材料，大幅缩短参赛准备周期。
